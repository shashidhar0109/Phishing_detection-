Prerequisites Check:
 1. Check if Docker is installed
docker --version

 2. Check if Docker Compose is installed  
docker compose --version

 3. Check system resources (recommended: 8GB+ RAM)
free -h

Step 1: Navigate to the Submission Folder

Navigate to the PS02_Docker_Submission folder
cd /path/to/PS02_Docker_Submission

Verify you're in the right directory
ls -la

You should see: docker-compose.yml, Dockerfile.backend, Dockerfile.frontend, etc.


Step 2: Quick Start (Automated)

Make the script executable
chmod +x QUICK_START.sh

Run the automated setup
./QUICK_START.sh

OR Manual Setup:

Step 2: Manual Setup (Alternative)

1. Start all services
docker compose up -d

2. Wait for services to initialize (2-3 minutes)
sleep 60

3. Check container status
docker compose ps

4. Check logs if needed
docker compose logs -f



Step 3: Verify System is Running

Check all containers are healthy
docker compose ps

Expected output should show all services as "Up" or "healthy":
 - finaltry-db-1 (PostgreSQL)
 - finaltry-redis-1 (Redis)  
 - finaltry-backend-1 (FastAPI)
 - finaltry-frontend-1 (React)
 - finaltry-celery-worker-1 (Background tasks)
 - finaltry-celery-beat-1 (Scheduled tasks)


Step 4: Access the System
 Open your browser and go to:
 Frontend Dashboard: http://localhost:3002
 Backend API: http://localhost:8001
 API Documentation: http://localhost:8001/docs


Step 5: Test the System

Test API connectivity
curl http://localhost:8001/api/stats

Test frontend
curl http://localhost:3002



if any issues troubleshooting commands are given below 

Troubleshooting:

If containers fail to start:
docker compose logs

If database connection issues:
docker compose restart db

If frontend not loading:
docker compose restart frontend

If API not responding:
docker compose restart backend

Full reset (removes all data):
docker compose down -v
docker compose up -d
